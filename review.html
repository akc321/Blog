Tools Used - Benefits and Drawbacks

The tools which we used on the current project include:

Trello

Trello is an organisation tool which uses cards with tasks assigned to them. It is very good to use on small projects and small group projects. The issue with Trello is that it can become very convoluted. If you move finished tasks to a global finished tasks card. It can be difficult to figure out how many tasks have been completed from each individual card. If you have a finished task card for each card holding tasks, you will quickly be overwhelmed by the amount of cards that you have. Trello’s value as an organisational tool quickly diminishes due to the difficulty of being able to assess how far you are in the project from viewing the project screen.

Other software such as Teamwork uses a ticked list and has similar functionality to Trello. But it is much simpler to see how far you are with a project as it makes it clear with a better visual depiction and better reporting elements. It also shows how many tasks have been completed against the amount of tasks you began with in a simple line of text.

For this project creating a website, Trello has been my preference over Teamwork, if it was a larger website I may change my mind. My organisation space has not become too convoluted and it is clear whether I have completed the task or not. I have adopted an unorthodox approach to using Trello setting a single task checklist on each task. This gives a visual cue on whether the task has been completed or not, without having to move it to a completed card. This makes it simple to see if all the tasks for a given objective have been completed.

Code Pen

Code Pen is a useful tool for web development, it can give you insight into how a particular feature has been coded. This insight is invaluable when learning how to do something new. Using Code Pen I have been able to understand the theory of how CSS Parallax scroll works, by being able to manipulate the data on the webpage and see instant results, you can get a better understanding of the role each element plays within a function. My only drawback to Code Pen is when using your laptops trackpad or mouse. If you accidentally hit the ‘back’ button or perform the same function using two fingers on the trackpad. You lose all data that you have written. This can be very frustrating especially if you are testing the waters of a function and are unsure how you got to the point that you were at.

Visual Studio

Visual Studio is a great Integrated development environment, I have been unable to take advantage of the add on functionality but using it in it’s most basic form has helped me to create my website quick and efficiently. The colour scheme aids with differentiating the different types of information. Whether it be a variable, function, object, array or any other element. It makes it simple to see what you are doing due to a beneficial colour scheme.

The Intellisense feature is very useful especially as a beginner to coding. It allows for mistakes to be easily seen so that they can be corrected. I also like the indent feature which displays a vertical line, on this project it has helped to make my containers easy to understand, as you can see exactly what is within them due to the vertical line stating at the start of a div tag and ending at the closure of the div tag (or any tag you wish to use).

Appropriateness of tools used

Trello

As mentioned above, Trello is good for smaller projects, due to this project being quite small it was very useful for organising the project and being able to see my progress with a quick look at the screen. The added features on Trello were useful to show if I had completed a task. I decided against using checklists (with multiple tasks) within tasks as you can not get a very good visual representation of these looking at the task on the whole. As you have to click into each individual task to see the tasks that have been set. I believe that Trello was the best tool to my knowledge for this task.

Code Pen

Code Pen was extremely useful for creating this website, There were many times that I needed to understand how a piece of code worked. By getting this understanding quickly by looking at other examples, I was delayed less and able to continue the creation of my website.

Visual Studio

Visual studio helped me to code my website efficiently and effectively, I was able to code my website very cleanly and be able to understand the data better by the colour scheme that Visual Studio displays. Along with Intellisense I was able to code with less unnecessary bugs due to input errors.

I have also added a hover function which highlights the tab which is being hovered over. Adding a class to all elements makes it simple to add elements as you only need to make changes to the HTML.

The last item that I am pleased about is keeping my website responsive allowing it to work on various sized screens. I am especially proud of the home screen and keeping the text within a certain area of the background image using Flexbox.

Productivity, efficiency and teams

On this project we performed our work as individuals. This meant that the methods for maintaining productivity and efficiency are very different in my eyes to the methods that you would use when working in a team.

When working on your own with the knowledge that you will not have to work in a team at any point. There are many shortcuts that you can take to improve efficiency. Such shortcuts include not having to set times on tasks to be completed for every task. This is especially true on small projects. This is due to the time it takes to set these times on each task against how much it will benefit you being nominal. You should try to only start a task when another task is complete so setting times on every task may not be effective as you may complete a task early. This is true for a team as well but by setting times for them to complete a task, you can make sure that you are working in unison and starting tasks without the information you need to complete a task.

Note: I do not believe an app exists which lets you set a time each task will take and once you have marked it as completed, will let you know if you are in front or behind in your project. An app like this would work for both individuals and teams and make sure that you are performing your task on time without having to reset when to start tasks and when they need to be completed.

Communication

One of the main differences between working as an individual and working in a team is communication. When defining a task it is crucial to understand the availability of the team and the skills that the team have. This allows you to properly delegate tasks to individuals by skills that they posses.

Visual Communication

Another crucial part of communication within teams is displaying when tasks have been completed. This can be as simple as moving a task on to a complete card in Trello or marking it as complete using the checklist function on Trello. This allows the team to know how far you are into the task, so that they can perform tasks when they are required to. Some tasks require other tasks to be completed before they can start. By adding reminders on Trello and also setting reminders this can be done.

Meetings

If working in a team I would not just rely on Trello to organise the team. By having small meetings, it will make sure people properly understand what they are doing and whether they are keeping on top of it. This will make sure that people do not become demotivated or unsure of their role within the project.

Informing Team Members Of Issues

Using Trello or another form of communication we can inform team members of issues as they happen. Depending on the severity of the issue a decision can be made whether to take team members away from there set task to help on the issue or whether to have an emergency meeting on ways to solve the issue. An issue can effect the whole project, so by communicating the issue to the team, it can be dealt with properly so that it has minimal effect on the project.

Team work

If I was to complete this project in a team, I would have assigned tasks to each individual. I would have assigned the task of creating the content to one or many individuals depending on the size of my team. Each would have their own section of the content to research and complete, this would be organised on Trello so that every member would understand what their task is and what other team members tasks are too.

I would set another individual with the task of creating the style of the website using dummy HTML data at first as the HTML content would not be available straight away.

I would organise the project making sure all team members are happy and motivated. I would make sure the team were using AGILE practices. I would keep a track on members using Trello and also by speaking to them and getting a better understanding of their ideas. This would allow me to give them direction if I believe what they are doing is moving away from the objective of the project.

I would then research how to make the project better in my spare time. Depending on the time set we could implement improvements once properly researched and the benefits of implementing outweigh the drawbacks of the time it takes to implement. This is all theoretical however as how I would approach the task would depend on the size of the team and the time limit set. I would use this approach above if we had a team of 3+ and if we had the time limit we currently have set.

Ways To Improve Productivity And Efficiency

To improve productivity and efficiency, I believe that planning is crucial and the amount of time you spend on planning is crucial. It will always depend on the task size and the time you have to complete that task. If the project is small there is no point in spending excessive time on planning. If your brain can compute the limited amount of tasks on the project you may not need to plan at all. It would be more efficient to dive straight into the project and start creating. However if a project has more than 4 or 5 small tasks I would advise to make a plan.

Why plans are effective

Creating a plan is useful for many reasons. If it is a plan to do with a project which you have a strong emotional connection to, It will move these ideas to the analytical part of the brain. This will be done by writing out a plan on either a web app such as Trello or if you put it on paper. Losing this emotion allows for better rational decisions to me made based on the specifics of the project.

Putting a plan on Trello can help for the project to be organised and kept on top of. If someone was falling behind or has missed a task, this can be seen and rectified depending on how the team leader wishes to deal with the issue.
